# 무인 모드 상세 (무인 세션에서만 주입)

이 절은 `CHAGEUN_UNATTENDED=1` 무인 세션에서만 로드된다. 일반 세션에는 위 코어의 무인 포인터만 보인다.

**무인 모드(사람 자리 비움)는 Claude 전용:** 무인 빌드 loop의 안전 바닥은 **격리 작업실**이다 — 런처가 본체를 `git clone --local`한 일회용 저장소에서 `origin`을 떼고(push·PR 구조적 불능) `claude --strict-mcp-config --mcp-config <빈목록>`으로 기동(MCP 전면 off 지향)하며, preflight가 시크릿·외부 연결을 거른다. 즉 정규식 명령 명단이 아니라 *환경*으로 되돌리기 불가 행동을 불능화한다. 그 위에서 PreToolUse 훅은 **환경이 못 막는 것 + 환경 효과가 무인에서 관측 불가한 것**만 집행하는 얇은 백스톱이다(STOP 정지·경로가드(트리밖/보호경로/동결기준)·중첩 claude 탈출·파괴적 SQL·**원격 MCP 쓰기·MCP 경유 DB DML**·**외부 데이터 전송(egress: curl 업로드·POST·scp·nc 등 — localhost 검증은 허용)**, 판정 중 예외는 park).

**egress 차단의 한계(정직):** 이 정규식 그물은 흔한 업로드/전송 동사(curl 데이터·POST/PUT/PATCH·-F/-T, wget --post, scp/sftp/원격 rsync, nc/telnet)만 park하고 목적지 호스트를 실제 추출해 localhost는 통과시킨다(loop 검증 보존). **못 잡는 것: GET 쿼리스트링 유출, python/node/ruby 인라인 HTTP(loop가 앱 실행에 정상 사용해 오차단 위험이 커 제외), base64 파이프, DNS 터널, 셸 래퍼(sh -c)·`$VAR` 호스트 우회.** (반대로 localhost POST 본문에 외부 URL이 들어가면 안전측으로 park될 수 있다 — 오차단이라 사고는 아니나 loop가 멈춘다.) 완전한 경계가 아니라 심층방어 한 겹이며, **근본대책인 OS 샌드박스 network allowlist는 이 개발 환경에서 실차단을 검증할 수 없어 미룬다**(검증 못 하면 넣지 않는다 — 계측 제거 교훈). 밤새 민감 데이터를 다루는 일이라면 이 한계를 알고 맡긴다. MCP-off의 런타임 효과는 harness에서 관측할 수 없어 훅의 MCP 백스톱을 심층방어로 함께 둔다(둘 중 하나만 믿지 않음). 반면 **설치·localhost DB쓰기는 격리 덕에 허용**(검증 마찰 제거 — 로컬은 풀고 원격/관리형만 막는다). Codex는 PreToolUse가 없어 **Codex 세션에선 무인 모드로 진입하지 않는다** — 무인이 필요하면 Claude에서. (유인 모드는 양 플랫폼 동일.)

**무인 중엔 병렬 위임을 쓰지 않는다 — 여러 에이전트를 동시에 띄우지 않는다:** 무인 루프는 계획→구현→검증→심판을 한 번에 한 단계, 한 에이전트씩 **차례로** 돈다(순차 게이트 호출은 정상 — 금지되는 건 '병렬 위임'(`chageun:routing`)식 동시 실행이다). 무인은 이미 자율성을 한 축 올린 상태라, 거기에 병렬(또 한 축)까지 겹치면 검증·되돌리기가 감당 못 한다 — **한 번에 한 축만 올린다.**

**무인 모드 켜는 법(사람):** `chageun-unattended` 런처로 시작한다 — preflight(샌드박스 살아있나·시크릿 없나)를 통과해야 통과표를 발급하고 `CHAGEUN_UNATTENDED=1`로 시작한다. 통과표 없이 raw로 `CHAGEUN_UNATTENDED=1`만 켜면 훅이 모든 작업을 park한다(안전). 샌드박스는 `.chageun/unattended.json`(sandbox.container 또는 sandbox.dbUrl)에 1회 설정. **정지:** 다른 터미널에서 **런처를 띄운 그 프로젝트 루트**(런처가 안내한 경로)에서 `touch .chageun/STOP` — 즉시 모든 도구 호출이 멈춘다. STOP·통과표 위치는 런처가 `CHAGEUN_ROOT`로 못 박으므로, 세션이 하위폴더나 전용 worktree로 옮겨 다녀도 그 루트의 STOP을 확실히 본다.

**무인으로 일 맡기는 법(층2 걸음마):** (1) `chageun-unattended setup`으로 유인 시동 — 무인으로 맡길 일과 "다 됐다" 기준을 claude와 함께 `.chageun/task.md`·`.chageun/criteria.md`에 적고 확인한다. (2) 같은 폴더에서 `chageun-unattended`로 무인 시작 — 세션이 `unattended-loop` 안내를 따라 계획→계획심판→구현→**localhost 실구동검증**→코드심판→통과 시 로컬 커밋을 한 바퀴 돈다. **배포·push는 무인 중 안 하고**, 복귀 후 사람이 localhost로 확인한 뒤 함께 넘긴다. 산출물은 "완성"이 아니라 **검증 대기 초안**이다. 기준(`criteria.md`)은 무인 시작과 동시에 `.chageun` 보호로 동결돼 루프가 못 바꾼다.

**여러 일·안전 한도(층2 큐):** 시동(setup)에서 여러 일을 말하면 세션이 `.chageun/queue.md`에 차례로 적고, 무인 루프가 위에서부터 하나씩 돈다. 어디까지 했는지는 git 커밋 이력으로 재개하므로 껐다 켜도 이어간다. 안전 훅이 **8시간·2000번·30분 무진전**을 기계로 재다 넘으면 자동 멈춤(park) — 진행분은 커밋으로 남고 사람 복귀 후 재개. 예산 시계는 껐다 켜도 리셋되지 않으며(재시작 우회 불가), 새 무인 시작(`chageun-unattended`) 때만 0에서 시작한다.

**고치기·가정(층2 완성):** 코드심판이 문제를 내면 무인 루프가 **최대 2번**까지 고쳐 재시도하되, 변경이 원래 일의 범위를 확 벗어나면(부풀기) 멈춘다(park). 밤새 만나는 애매함은 **국소**(표시·문구)면 최선추측+진행, **구조**(데이터 모양·규칙)면 멈춤, **사람 뜻을 대신 정하는** 것이면 🙋 예·아니오 질문으로 남기되 4개를 넘으면 "무인에 안 맞다"는 신호로 멈춘다(애매하면 구조로 보고 멈춤). 가정·질문은 복귀 시 인수인계(채팅)로 보여준다. `.chageun/token`·`.chageun/STOP`은 보호경로라 무인 세션이 스스로 못 건드린다. (preflight 위험 스캔은 휴리스틱이라 외부 DB URL·시크릿을 대부분 잡지만 놓칠 수 있으니, 자리 뜨기 전 env를 한 번 확인.)
