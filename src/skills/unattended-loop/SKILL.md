# 무인 빌드 loop (unattended-loop) — 걸어다니는 뼈대

사람이 자리를 비운 사이, 이 세션이 **할 일 하나**를 받아 파이프라인을 한 바퀴 돌아 **검증 대기 초안**을 만든다. "완성"이 아니라 "사람이 돌아와 로컬로 열어 검증할 초안"이다.

## 발동 전 자기가드 (기계 바닥이 없으면 멈춤)

이 스킬은 **무인 안전 훅이 켜진 Claude 세션**에서만 안전하다. LOOP를 시작하기 전에 확인한다:
- 무인 표식(`CHAGEUN_UNATTENDED=1`)과 통과표가 있는가? 없으면 **LOOP를 시작하지 않고** 사람에게 "무인 모드가 아닙니다 — `chageun-unattended` 런처로 시작하세요"라고 알린다.
- Codex 등 PreToolUse 훅이 없는 환경이면 무인 LOOP 금지(안전 바닥 부재).

**이 자기가드는 심층 방어의 얇은 겉막이다.** 진짜 기계 바닥은 (a)런처가 `claude`를 하드코딩해 시작하고(codex로는 무인 진입 불가) (b)빌드가 이 스킬을 Codex 산출물에서 제거하는 것 — 이 둘이지 이 문단이 아니다.

## SETUP — 시동 (사람 있을 때, 유인)

무인 시작 **전**(아직 `.chageun`가 안 잠긴 유인 상태)에 사람과 함께 준비한다:

1. 사람이 채팅으로 할 일을 말한다.
2. 세션이 일을 적는다:
   - **일이 하나면** `.chageun/task.md`(할일 카드: 왜·무엇·범위·안 하는 것) + `.chageun/criteria.md`(성공기준).
   - **여러 일이면** `.chageun/queue.md`에 **차례로** 적는다 — 각 항목: 번호·제목 + 그 일의 성공기준(자동으로 잴 수 있는 형태). 큐는 위에서부터 처리된다.
   - 성공기준은 반드시 **자동으로 잴 수 있는 형태**(존재·텍스트·상태 **단언**, 로컬 실행 시나리오)로. "예쁘다" 같은 취향 기준은 부적합.
3. 두 파일을 사람에게 **보여주고**, 성공기준은 **한 줄씩** 짚어 "이 줄 맞나요?"로 확인받는다(대충 넘기지 못하게). 통째로 "네" 받고 넘기지 않는다 — 느슨한 기준이 그대로 동결되면 루프가 자기 기준을 스스로 통과시켜 커밋한다.
4. 확인되면 사람은 `chageun-unattended`로 무인을 시작한다. 무인이 켜지면 `.chageun/` 전체가 쓰기 금지(보호)라 `criteria.md`가 **동결**된다 — 루프가 스스로 기준을 못 바꾼다.

## 적합 확인 (한 번, 무인 시작 직후)

LOOP 전에 이 일이 무인에 맞는지 기계적으로 점검한다. 하나라도 아니면 **park**하고 사람을 기다린다:
- 기준(`criteria.md`)이 **단언으로 잴 수 있나**(사람 눈 판정이 핵심이 아닌가).
- 샌드박스가 살아있나(preflight가 이미 확인 — localhost 서버·DB).
- 디자인·취향·새 사업결정이 산출물의 핵심이 **아닌가**(핵심이면 "구조만, 스타일은 복귀 후"로 쪼개거나 park).

## LOOP — 한 바퀴 (할 일 1개)

**여러 일이면(큐):** `.chageun/queue.md`를 **위에서부터** 하나씩 꺼내 아래 한 바퀴를 돈다. 한 항목이 green이면 완료 커밋 메시지에 **항목 표식**(번호·제목)을 넣고 다음으로. **재개(껐다 켜도):** 어디까지 했는지는 **git 커밋 이력**으로 판정한다 — 이미 완료 표식이 커밋된 항목은 건너뛴다(별도 진행파일 없음).

**한도(안전 브레이크):** 안전 훅이 시간(8시간)·작업량(2000번)·헛돎(30분 무진전)을 기계로 재다가 넘으면 자동 park한다. 진짜 상한은 **시간·작업량**이고, 헛돎(워치독)은 커밋으로 리셋되는 **보조 브레이크**다. park되면 더 진행하지 말고 FINISH로 간다(진행분은 커밋으로 남아 재개됨). **예산(시간·작업량)으로 park되면 Write 도구도 막히므로, 인수인계(park 목록·대기표)는 파일이 아니라 채팅 텍스트로 남긴다.**

1. **계획:** `task.md`·`criteria.md`(또는 큐 항목)를 읽고 짧은 구현 계획을 세운다.
2. **계획심판:** `plan-validator` 에이전트를 Agent 도구로 부른다. NO-GO면 **park**.
3. **구현:** `code-implementer` 에이전트를 부른다(무인 금지 안내는 그 정의에 있음). BLOCKED/NEEDS_CONTEXT면 **park**.
4. **로컬 실구동검증:** 앱을 **localhost(샌드박스)**로 띄워 `criteria.md`의 **단언**을 판정한다. 스크린샷은 판정 근거가 아니라 복귀 후 사람 검증용 **증거물**로 남긴다. **배포하지 않는다.**
5. **코드심판:** `pr-reviewer` 에이전트를 부른다. blocker가 나오면 **고치기 1회(N=1)**: 구현→검증→검토를 한 번 더. 그래도 실패면 **park**.
6. **통과 시 저장:** 두 심판 green + 기준 단언 통과면 `git commit`(로컬). **push·배포는 하지 않는다**(훅도 막는다).

무인 중 되돌리기 비싼 것(push·배포·운영 DB 쓰기·삭제·외부 전송)은 **절대 하지 않고** 전부 park한다. 구조를 바꾸거나 사람 의도를 대신 정해야 하면 park한다.

## FINISH — 마무리 → 사람 대기표

복귀한 사람이 볼 것을 남긴다:
- 🙋 물어볼 것(있으면, 상한 3~4개; 넘으면 적합 판정이 틀린 신호 → park).
- **park 목록**(사람 결정이 필요해 멈춘 것들 + 이유).
- 스크린샷 증거 + **로컬로 여는 법**("이 한 줄로 켜세요 / 여기 눌러 확인").
- **hard-stop 대기표** — "이제 push/배포 할까요?"는 여기 적고, **배포는 사람이 복귀해 localhost로 확인한 뒤 함께** 넘긴다.
- 한 일 기록(무엇을·왜, self-heal 있었으면 그 이력).

산출물은 **"검증 대기 초안"**이다. "완성"이라 말하지 않는다 — 사람의 검증 근육을 지킨다.
